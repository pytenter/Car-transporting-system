# 新能源车队调度仿真项目：小白运行指南与答辩问答

本文面向**零基础或基础较弱**的同学：先说明**怎么把项目跑起来**，再用**老师常问问题**的形式串起技术要点。回答时不必背代码，理解**“系统在做什么、为什么这样设计”**即可。

---

## 一、项目是做什么的？

用计算机**模拟**一支**新能源物流车队**在城市路网里接单、行驶、充电、送货的过程。

- **输入**：随机生成的城市图、任务（取货/送货点、释放时间、截止时间、重量）、车辆（载重、电池、速度、耗电）、充电站（桩数、充电功率）。
- **输出**：每种**调度策略**完成多少单、是否超时、总里程、充电等待、综合**分数**；可选生成 **JSON 结果**、**Markdown 对比报告**，以及 **Web 仪表盘**里的路线回放与图表。

可以把它理解成：**数据结构课里的“图上的资源调度 + 时间推进仿真”**，外加**启发式/智能体**做决策。

---

## 二、环境要求

- **Python 3.10+**（你当前若已是 3.12 也可以）。
- **默认不需要安装第三方库**：动态仿真 + Web 仪表盘走标准库即可（`http.server` 等）。
- **可选**：若要做**静态精确求解（CPLEX）**与动态结果对比，需自行安装 `docplex`、`cplex` 及有效许可证；没有 CPLEX 时，**只跑动态部分完全没问题**。

## 三、如何运行（请务必在项目根目录执行）

**项目根目录**：指包含 `main.py`、`dashboard.py` 的那一层文件夹（例如 `Datastructrure--project-master`）。

在终端里先进入该目录，再执行下面命令。若在别的目录运行，可能出现 `can't open file` 或结果写错位置。

### 1. 推荐：先建虚拟环境（可选但专业）

```bash
python -m venv .venv
# Windows PowerShell：
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

### 2. 跑一轮“小规模”对比（最快验证项目正常）

不调用 CPLEX，只跑动态策略：

```bash
python main.py --no-oracle --scales small --output results/summary_demo.json
```

你会在终端看到一张表：`scenario / strategy / completed / score` 等；并生成 `results/summary_demo.json` 和 `results/comparison_report.md`（若当前工作目录就是项目根）。

### 3. 跑完整规模（small + medium + large）

```bash
python main.py --no-oracle --scales small medium large --output results/summary_dynamic.json
```

### 4. 启动 Web 可视化仪表盘

```bash
python dashboard.py
```

浏览器打开：**http://127.0.0.1:8765**（若端口被占用，以终端提示为准）。

仪表盘可做：路线回放、天气模式切换、加载 `results` 下的 benchmark JSON 等。详细说明见同目录 **`README.md`**。

### 5. 天气统计预计算（仪表盘加载更快，可选）

```bash
python precompute_weather_stats.py --seed 20260309 --exact-time-limit 120 --exact-mip-gap 0.0
```

会生成或更新 `results/weather_stats.json`（若本机无求解器，脚本内应有降级逻辑；若报错可看终端提示，不影响仅用动态仿真）。

### 6. 带 CPLEX 的“动态 vs 静态”全量对比（仅在有许可证时）

```bash
python main.py --allow-collaboration --exact-backend cplex --exact-scales small medium large --output results/summary_cplex.json
```

无 CPLEX 时会出现静态行 `FAILED` 或跳过，**动态行仍然有效**。

## 五、核心流程：仿真是怎么“推时间”的？

主循环在 `FleetSimulator.run` 中，大意是：

1. **离散时间** `now = 0, 1, 2, …` 推进到仿真结束时刻。
2. 每个时刻把**已到释放时间**的任务放进待调度列表 `pending`。
3. 找出当前**已空闲**的车辆（`available_time <= now`）。
4. 在车库且电量偏低的车辆可先触发**空闲补电**逻辑。
5. 在仍有待派单且有空车、且电量高于**安全阈值**时，调用策略的 `choose(...)`，得到“派哪辆车（或多车协同）执行哪个任务”。
6. 对选中的任务做**路径与耗时规划**（最短路、是否中途充电、充电站**端口排队**等），更新车辆状态与完成时间。
7. 根据完成时间计算该任务得分，记入 `completed`；可选记录 `DispatchEvent` 供前端回放。
8. 若任务未派完或车辆仍忙，时间继续走；**未服务任务**在最后用**惩罚**扣进总分。

**一句话答辩版**：我们是**事件驱动的离散时间仿真**，每一步用**图最短路**算路程，用**充电站多端口时间线**模拟排队，策略只负责**在可行方案里选任务-车辆组合**。

## 六、用到了哪些“数据结构课”知识？

| 知识点 | 在项目里哪里体现 |
|--------|------------------|
| **图（邻接表）** | `WeightedGraph` 存节点与边 |
| **最短路** | `shortest_path`：Dijkstra + **优先队列（heapq）** |
| **排序与扫描** | 任务按 `release_time` 排序后用指针 `cursor` 批量释放 |
| **列表/字典** | 待派任务、车辆表、充电站、事件记录 |
| **贪心/启发式** | 最近任务优先、紧急度加权等策略 |
| **元启发式** | 模拟退火策略 |
| **多臂老虎机思想** | 超启发式 UCB 在子策略间选择 |
| **强化学习雏形** | 简单 Q 表更新（离散状态-动作） |
| **数学规划（拓展）** | 静态 baseline：MIP，需外部求解器 |

老师若问“最难的数据结构是什么”，可以答：**在图上反复最短路 + 充电站端口的“多队列/最早可用时间”维护**，既要快又要符合物理约束。

## 七、充电站排队是怎么简化的？

`ChargingStation` 为每个**端口**维护 `_port_available_times`：表示该端口何时空出来。车辆到达时：

- 选**能最早开始充电**的端口；
- 开始时间 = max(到达时刻, 端口空闲时刻)；
- 更新该端口的结束占用时间。

这就是一种**离散时间资源预约**，比真实充电桩规则简单，但足以体现**排队与利用率**对调度的影响。

## 八、分数怎么算？（老师爱问“目标函数”）

单任务得分（与静态求解里口径一致）大致为：

- 基础项：希望响应快、路程短、少排队  
- **超时**再减去 `overtime_penalty`  
- **未完成**任务按个数乘 `unserved_penalty` 从总分里扣  

动态仿真里具体系数在 `FleetSimulator._score_task` 中写死（如响应时间、距离、充电等待的权重）；**分数越大越好**（在对比表里通常按 `score` 排名）。

## 九、有哪些调度策略？各一句话**

| 注册名 | 中文直觉 |
|--------|-----------|
| `nearest_task_first` | 空闲车里选离任务最近的组合（贪心） |
| `max_task_first` | 偏重“大单/紧任务”等特征（大任务优先类） |
| `urgency_distance` | 紧急度与距离综合权衡 |
| `auction_multi_agent` | 拍卖/竞价式多车分配思想 |
| `metaheuristic_sa` | 模拟退火在候选决策上搜索 |
| `reinforcement_q` | 简易 Q-learning 学习派单偏好 |
| `hyper_heuristic_ucb` | 超启发式：用 UCB 在多个子规则间动态选择 |

答辩时不必展开公式，说清：**都是在线启发式，在“信息逐步到达”的条件下做决策**即可。

## 十、动态 vs 静态（全信息 CPLEX）区别**

- **动态仿真**：时间一步步走，任务按释放时间出现，更像真实运营。
- **静态精确解**：假设一开始就知道全部任务，用混合整数规划求（近似）最优分配与顺序；算力要求高，且本项目里充电排队与动态的建模口径**不完全相同**（报告里也有说明）。

没有 CPLEX 时，可以说：**我们用多种动态策略横向对比，静态解作为可选理论下界/对照**。

---

## 十一、协同任务（`--allow-collaboration`）是什么？

当**单车载重不够**时，若开启协同，允许多车**共同服务一个超重任务**。仿真里把任务重量按车辆容量**比例分摊**到各车，路径与能耗仍按各自行驶计算；用于展示**协作覆盖可行性**与可视化。

---

## 十二、随机种子与可复现

`main.py` 默认 `--seed`（如 `20260309`）控制图、任务、车辆随机性。**同一 seed、同一参数**，多次运行动态结果应对齐（策略若含随机性，也会用派生 seed）。答辩时说：**保证对比公平、实验可复现**。

## 十三、老师高频问答（简答模板）

**Q：你们问题建模成什么数学对象？**  
A：城市路网是**无向加权图**；调度是**带时间窗、容量、电量约束的车辆路径与任务分配**的在线决策问题；静态版写成 **MIP**。

**Q：最短路用什么算法？复杂度？**  
A：**Dijkstra**，用**最小堆**；单次近似 \(O((V+E)\log V)\)，路径结果有缓存减少重复计算。

**Q：为什么不用纯深度学习？**  
A：课程项目侧重**可解释、可复现**的仿真与经典启发式；深度模型需要大量训练数据与工程成本，且不如规则策略便于写报告对比。

**Q：仿真时间步长是 1 代表什么？**  
A：是**离散时间单位**（与场景 `horizon` 等单位一致），便于教学实现；真实系统可解释为“分钟”等，只要全文一致。

**Q：结果 JSON 里 `mode` 是什么？**  
A：`dynamic` 表示在线仿真；`static_exact_*` 表示静态求解器一行对照；带 `reduced` 表示因 **CPLEX 许可证规模限制**对任务/车辆子集做了缩减。

**Q：你个人负责哪部分？**  
A：按实际分工答：例如“策略实现”“仿真主循环”“前端可视化”“实验脚本与报告”等（勿夸大）。
